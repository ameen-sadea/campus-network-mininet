# 《基于 Mininet 的校园网构建》演讲稿

预计时长：约 5 分 35 秒  
对应文件：`校园网项目汇报.pptx`

## 第 1 页：封面（约 20 秒）

大家好，我们是第十二组。我们这次的项目是基于 Mininet 构建校园网。

这次我们希望完成的不只是一张能够 Ping 通的网络拓扑，而是一套包含分层架构、VLAN 划分、安全边界、VPN 接入和运行观测的完整校园网仿真。接下来我会重点介绍整体框架、IP 分配策略，以及我们认为比较有价值的几个亮点。

## 第 2 页：项目定位（约 40 秒）

先简单介绍项目规模。整个系统包含 2 台核心交换机、4 台汇聚交换机、18 台接入交换机，以及 31 台主机和服务器。

我们首先完成了课程要求中的 VLAN 隔离、跨网段路由、服务器访问和 ACL 控制。在此基础上，又加入了主分校区互联、模拟公网、DMZ 隔离区和 GRE VPN。

最终形成的不是单纯的拓扑展示，而是一套可运行、可验证、可观察的系统。前端看到的路径和指标，都来自后端真实执行的网络操作。

## 第 3 页：整体架构（约 60 秒）

整体网络采用经典的核心、汇聚、接入三层架构。

接入层直接连接宿舍、办公、教学、人事、财务和服务器等终端，并通过 Access 端口把终端加入对应 VLAN。汇聚层按照地理位置和业务类型进行区域收敛，例如办公教学区、生活无线区和数据中心区。核心层则负责全网流量的集中交换，同时连接第二校区。

在传统三层架构之外，我们增加了独立的边界防火墙 fw1。它分别连接校园内网、DMZ 和模拟公网，形成三臂防火墙结构。

这样设计最重要的一点，是所有跨安全区域的流量都必须经过 fw1。也就是说，前端展示的防火墙路径和底层真实数据路径是一致的，不会出现“图上经过防火墙，实际流量却绕开”的问题。

## 第 4 页：IP 与 VLAN 分配（约 65 秒）

IP 地址采用按业务域和安全域分配的思路。

校园内网统一使用 10.0.0.0/8 地址空间，再为每个业务区域划分独立的 /24 子网。为了便于理解和管理，我们让 VLAN ID 与 IP 地址第三段尽量对应。例如办公区使用 VLAN 20，对应 10.0.20.0/24，网关统一设置为 10.0.20.1。

人事和财务分别使用 VLAN 50 和 VLAN 60，服务器区使用 VLAN 100。这些网段划分之后，可以直接作为 ACL 的匹配条件。

不同安全区域使用不同地址空间：DMZ 使用 10.0.110.0/24，模拟公网使用文档保留网段 203.0.113.0/24，GRE 隧道内部只需要两个地址，因此使用 172.16.0.0/30。

这种规划让地址本身带有业务含义，既方便编写路由和安全策略，也方便测试时快速定位问题。

## 第 5 页：核心转发与安全机制（约 65 秒）

在实现上，我们采用了一个轻量化的混合架构。

OVS 负责二层交换。连接终端的端口配置为 Access，交换机之间配置为 Trunk，从而完成 VLAN 的二层隔离和传递。

跨 VLAN 路由则由 Linux 内核完成。我们创建 Veth Pair，把 fw1 和 Linux root 命名空间连接起来，再为每个 VLAN 创建 802.1Q 子接口，配置对应的 .1 网关，并开启 IP Forward。

这样，OVS 专注于它擅长的二层转发和 OpenFlow ACL，Linux 则使用成熟的原生协议栈完成三层路由。

安全控制也分为两层：iptables 负责有状态防火墙，例如允许已建立连接的返回流量、阻止外网主动访问内网；OVS 流表负责按照源地址、目的地址和优先级精确阻断违规流量。

## 第 6 页：项目亮点（约 85 秒）

我们重点总结四个亮点。

第一个是三臂防火墙和 DMZ。公开网站和 VPN 服务放在 DMZ 中，外网可以访问指定的公开服务，但是不能直接进入校园内网；即使 DMZ 主机被入侵，也会被禁止向内网横向移动。

第二个是 GRE VPN 安全接入。访客在物理上接入公网区域，开启 VPN 后，访问内网的流量通过 GRE 隧道进入，并继续受到 ACL 权限控制。关闭 VPN 时，公网物理网段无法直接访问内网，属于 fail-closed 的设计。

第三个是双层防护。我们同时使用 iptables 有状态策略和 OVS 高优先级流表，解决了 ICMP 可能复用连接跟踪状态的问题，也让边界控制更加可靠。

第四个是前后端真实联动。用户在网页上执行 Ping、切换 VPN 或进行服务访问时，后端会真正执行对应操作，再把延迟、丢包、路径、吞吐和策略命中结果返回前端。拓扑图因此不只是示意图，而是一个轻量的网络数字孪生界面。

## 第 7 页：总结（约 40 秒）

最后总结一下。

我们从三层架构和地址规划开始，完成了 VLAN 隔离和跨网段路由；随后加入 DMZ、边界防火墙、ACL 和 VPN，让网络具备较完整的安全边界；最后通过 REST API 和可视化面板，把真实运行状态展示出来。

如果说课程的基本要求是把网络搭起来，那么我们这次进一步完成的是：把校园网设计出来、运行起来，并且让它能够被验证和观察。
