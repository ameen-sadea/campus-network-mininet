================================================================================
第一部分  环境安装
================================================================================

你需要两样东西：
  ① Windows 上的 Python（用来开网页）
  ② WSL 里的 Ubuntu + Mininet（用来跑真网络，可选但推荐）

只装 Python、不装 WSL 也能演示。右上角会显示「演示就绪」，走本地模拟，前后端分离。
装了 WSL 并连上 API，右上角会变成「Mininet 已连接」，测试结果是真 Ping/真 HTTP。


--------------------------------------------------------------------------------
步骤 1  安装 WSL（Windows 子系统 Linux）
--------------------------------------------------------------------------------

1. 右键「开始」→「Windows PowerShell（管理员）」或「终端（管理员）」。

2. 依次输入下面三条：

   dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
   dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
   wsl --set-default-version 2

3. 安装 Ubuntu：

   wsl --install -d Ubuntu

   如果提示已安装，可跳过。装完会自动弹 Ubuntu 黑窗口，让你设「用户名」和「密码」。
   密码输入时屏幕不显示字符，这是正常的，输完直接回车。

4. 确认是 WSL2：

   wsl -l -v

   看 VERSION 列是不是 2。若是 1，执行：wsl --set-version Ubuntu 2


--------------------------------------------------------------------------------
步骤 2  在 Ubuntu 里装 Mininet 和依赖
--------------------------------------------------------------------------------

1. 打开「终端」或 PowerShell，输入：

   wsl -d Ubuntu

   进入 Linux 后，提示符前面一般是你的用户名。

2. 更新并装 Mininet（第一次可能要几分钟）：

   sudo apt update
   sudo apt install -y mininet openvswitch-switch iproute2 python3

3. 启动 Open vSwitch：

   sudo service openvswitch-switch start

4. 验证能不能用（看到 mininet 版本号就行）：

   sudo mn --version

5. 验证项目文件能读到（把路径换成你电脑上的实际路径）：

   ls "/mnt/c/Users/你的用户名/Documents/作业/计算机网络作业/mininet/campus_network.py"

   能看到文件列表就说明 Windows 盘在 WSL 里访问正常。


--------------------------------------------------------------------------------
步骤 3  在 Windows 装 Python
------------------------------ wsl -d Ubuntu
1. 去 python.org 下载 Python 3.10 或更高版本，安装时务必勾选：
   「Add Python to PATH」（添加到环境变量）

2. 装完打开 cmd，输入：

   python --version

   能显示版本号就 OK。

3. 再确认：

   where python

   能显示 python.exe 的路径就 OK。


--------------------------------------------------------------------------------
步骤 4  记住你的 sudo 密码
--------------------------------------------------------------------------------

WSL 里第一次跑网络仿真会要你输入 Linux 用户密码（就是装 Ubuntu 时设的那个）。
演示时会在「MininetAPI」那个黑窗口里提示 [sudo: authenticate] Password:
你在那个窗口里输入密码、回车，（输入的密码不可见但是会生效）。

第二部分  一键启动（run_demo.bat）
================================================================================

项目目录结构（你关心的部分）：

  mininet\
    run_demo.bat          ← 双击这个，一键启动
    campus_network.py     ← 后端（bat 自动调，不用你管）
    frontend\
      index.html          ← 浏览器打开的页面


--------------------------------------------------------------------------------
启动步骤（按顺序做，不要跳）
--------------------------------------------------------------------------------

1. 打开文件夹：
   C:\Users\你的用户名\Documents\作业\计算机网络作业\mininet

2. 双击 run_demo.bat

3. 本窗口会依次显示：
   [1/4] WSL OK
   [2/4] Starting Mininet API ...
   [3/4] Waiting for API inside WSL ...
   [4/4] Opening browser ...

4. 同时会弹出两个新窗口，名字分别是：
   · MininetAPI    — WSL 里跑真网络 + API
   · FrontendHTTP  — Windows 里开网页服务（端口 8000）

5. 【重要】看 MininetAPI 窗口：
   如果出现 Password: 或 [sudo: authenticate] Password:
   → 在这个窗口里用键盘输入你的 Linux 密码（不显示字符）
   → 回车
   → 不要关这个窗口，等它跑完建网（大约 30～90 秒）

6. 主窗口 [3/4] 会每隔 2 秒跳一次秒数，API 就绪后显示「API ready」

7. 浏览器自动打开：
   http://localhost:8000/index.html?api=http://WSL的IP:5000

8. 演示结束后：
   分别关掉 MininetAPI 和 FrontendHTTP 两个窗口（或里面按 Ctrl+C）
   主窗口按任意键退出

--------------------------------------------------------------------------------
常见启动问题
--------------------------------------------------------------------------------

问题：MininetAPI 报 WSL not found
解决：回到第一部分，把 WSL 和 Ubuntu 装好。

问题：本窗口报 Python not found
解决：重装 Python 并勾选 Add to PATH。

问题：一直 Waiting... 超过 120 秒
解决：看 MininetAPI 窗口——多半是没输入 sudo 密码，或 Mininet 报错。
      在 MininetAPI 里输入密码后等它建完网。

问题：浏览器打开但是空白/目录列表
解决：手动打开 http://localhost:8000/index.html（带 index.html）

问题：右上角一直是「演示就绪」不是「Mininet 已连接」
解决：MininetAPI 窗口可能没起来；页面仍能演示 ACL/VPN/路径，只是本地模拟。


================================================================================
第三部分  先搞懂这张网（演示前 1 分钟口述）
================================================================================

这是一张「主校区 + 分校区」的校园网，三层结构：核心 → 汇聚 → 接入。

核心交换机 cs1 是「大脑」：
  · 负责不同 VLAN 之间的三层路由
  · 所有 ACL 防火墙规则只下发在 cs1 上（创新点：集中管控）

三类访客（答辩必讲）：

+----------+--------+------------------+----------------------------------------+
| 访客     | 接在哪 | 能干什么         | 图上怎么看                             |
+----------+--------+------------------+----------------------------------------+
| 访客-1   | sw13   | 只能和访客-2互访 | 左下访客 WiFi 区，紫色虚线 gst1↔gst2   |
| 访客-2   | sw13   | 同上，不能进内网 | sw13 没有上联线（物理隔离岛）          |
+----------+--------+------------------+----------------------------------------+
| 访客-3   | sw15   | 经 VPN 才能进内网| 左上 DMZ 区，和 VPN 服务器挨在一起     |
| (gst3)   | DMZ区  | 未连 VPN 不能进  | 天蓝虚线 sw15↔cs1 是 DMZ 核心直连      |
+----------+--------+------------------+----------------------------------------+

DMZ 区（图左上角，贴在 cs1 旁边）三个点：
  gst3（访客-3）、vpn_srv（VPN 服务器）、sw15（DMZ 交换机）

线型含义（图右上角图例）：
  · 实线白/灰 — 正常光纤/网线
  · 紫色虚线 — 访客 WiFi 同网段互访（不经过核心）
  · 天蓝虚线 — sw15 直连 cs1（DMZ 专用上行，创新点）
  · 靛蓝虚线 — gst3 ↔ vpn_srv 的 GRE VPN 隧道（创新点）

保密区：人事处（hr1/hr2）、财务处（fin1/fin2），接在 as1 下面。
  宿舍区不能访问；保密区也不能主动访问 DMZ、VPN、外网。

服务器区：web、ftp、dns 在 sw14，经 as3 上联。
  宿舍可以访问 web/ftp（ACL 允许）。

左侧 · 控制面板
--------------------------------------------------------------------------------

【源主机】下拉框（绿色边框）
  · Ping、HTTP、FTP 都从这儿选「谁发起」
  · 必须选，否则点测试会弹窗提示

【目标主机】下拉框
  · 只给 Ping 用，选「 ping 谁」
  · HTTP 固定访问 Web 服务器；FTP 固定访问 FTP 服务器，不用选目标

【测试 Ping】蓝色大按钮
  · 模拟/真实 ICMP 连通性
  · 结果在右下「测试结果」区，路径在拓扑图上高亮

【HTTP】橙色按钮
  · 测试源主机能否用 HTTP 访问 web（10.0.100.2）
  · 成功显示 [HTTP 200]；失败显示阻断原因

【FTP】青色按钮
  · 测试源主机能否从 ftp 下载 share.txt
  · 成功显示 [FTP OK]

【ACL】黄色按钮
  · 在拓扑图上高亮所有与 ACL 相关的链路（防火墙规则可视化）
  · 测试结果区会写 [ACL] 规则已高亮

【清除】灰色按钮
  · 去掉拓扑上的路径高亮、红色 ✕ 阻断标记、ACL 高亮
  · 测试结果区追加 [清除] 状态已清除（不清空历史文字，只清图）


--------------------------------------------------------------------------------
左侧 · ACL 防火墙规则列表
--------------------------------------------------------------------------------

作用：把 cs1 上的主要 ACL 用大白话列出来，红底=阻断，绿底=放行。

规则要点（和拓扑阻断现象一一对应）：
  · 访客WiFi → 内网：BLOCK（gst1/gst2 出不了 sw13）
  · 访客-3 → 内网：BLOCK（gst3 没开 VPN 时）
  · 访客-3+VPN → 内网：ALLOW（开了 VPN 后视同宿舍权限）
  · 访客-3 → VPN服务器：ALLOW（没 VPN 也能 ping 通 vpn_srv）
  · 访客-3+VPN → 人事处/财务处：BLOCK（VPN 不能提权进保密区）
  · 宿舍A/B → 人事处/财务处：BLOCK
  · 人事处/财务处 → DMZ：BLOCK（保密区禁止外出）
  · 宿舍区 → 服务器区：ALLOW


--------------------------------------------------------------------------------
左侧 · VPN 隧道
--------------------------------------------------------------------------------

【开关】
  · 只影响访客-3（gst3），不影响 gst1/gst2
  · 关：gst3 不能访问内网（除 vpn_srv 外），拓扑上在 cs1 画红 ✕
  · 开：gst3 经 GRE 隧道进内网，可访问办公楼、服务器等；仍不能访人事/财务

【状态文字】
  · 关：灰色点，「隧道未连接」类提示
  · 开：绿色闪烁点，「隧道已连接」

连上 Mininet 后，拨开关还会通过 API 同步到真网络的 gst3 路由和 cs1 流表。

测试后拓扑上的现象（务必会给评委讲）：

  成功时：

================================================================================
第五部分  推荐演示流程（循序渐进，约 8～10 分钟）
================================================================================

按下面顺序做，每一步都「操作 → 指拓扑 → 念结果」，逻辑最清楚。


--------------------------------------------------------------------------------
第 0 步  开场（30 秒）
--------------------------------------------------------------------------------

1. 双击 run_demo.bat，在 MininetAPI 窗口输入 sudo 密码。
2. 等浏览器打开，右上角变「Mininet 已连接」或「演示就绪」。
3. 指着拓扑说：核心 cs1 统一做路由和 ACL；左上 DMZ 是访客-3 和 VPN；
   左下 sw13 是访客 WiFi 隔离岛。


--------------------------------------------------------------------------------
第 1 步  正常内网通信（建立 baseline）
--------------------------------------------------------------------------------

操作：
  源主机选「宿舍A-1」(dA1)
  目标选「办公楼-1」(ofc1)
  点【测试 Ping】

现象：
  绿色路径，宿舍 → sw8 → as2 → cs1 → as1 → sw1 → 办公楼
  结果区 [成功]

口述：跨 VLAN 要经过核心 cs1 做三层转发，普通业务畅通。


--------------------------------------------------------------------------------
第 2 步  ACL 阻断宿舍访问保密区
--------------------------------------------------------------------------------

操作：
  源「宿舍A-1」，目标「人事处-1」(hr1)，点 Ping

现象：
  红色路径，在 as1 往人事方向链路上出现 ✕
  结果区 [阻断]

再测：宿舍A-1 → 财务处-1，同样阻断。

口述：人事、财务是保密区；ACL 在 cs1 执行，拓扑上在 as1 出口示意阻断。


--------------------------------------------------------------------------------
第 3 步  访客 WiFi 物理隔离（创新点）
--------------------------------------------------------------------------------

操作：
  源「访客-1」(gst1)，目标「访客-2」(gst2)，点 Ping

现象：
  紫色虚线 gst1↔gst2 高亮绿色，成功

再操作：
  源「访客-1」，目标「办公楼-1」，点 Ping

现象：
  路径到 sw13 就停，sw13 外侧链路红色 ✕
  结果区提示访客WiFi隔离

口述：gst1/gst2 在 sw13 隔离岛，没有上联，只能互访，这是物理隔离不是单纯 ACL。


--------------------------------------------------------------------------------
第 4 步  访客-3 与 DMZ、VPN（创新点，重点）
--------------------------------------------------------------------------------

先确认 VPN 开关是【关】。

操作 A：
  源「访客-3」(gst3)，目标选不到 vpn_srv 时用 Ping 目标栏——
  若下拉里没有 vpn_srv，可 Ping「办公楼」对比；先测 gst3 → vpn_srv：
  实际上 vpn_srv 在服务器类，Ping 目标选「VPN服务器」若列表有；
  没有则看 HTTP 前先 Ping 内网。

  源 gst3，目标 ofc1，VPN 关，点 Ping → 阻断，✕ 在进 cs1 处

操作 B：
  打开 VPN 开关，等状态变绿
  再 gst3 → ofc1 Ping → 成功，路径经 gst3—vpn_srv 靛蓝隧道再进内网

操作 C：
  VPN 保持开，gst3 → 人事处-1 Ping → 仍阻断

操作 D：
  源 gst3，目标 VPN服务器（vpn_srv），VPN 关也可 Ping → 成功
  （DMZ 区内可达，不经过内网 ACL 603）

口述：
  gst3 在 DMZ，sw15 天蓝虚线直连 cs1；
  没 VPN 不能进内网；开了 GRE 隧道才能进，但不能进保密区——
  体现「最小权限」和「VPN 不等于全网权限」。


--------------------------------------------------------------------------------
第 5 步  应用层 HTTP / FTP（创新点）
--------------------------------------------------------------------------------

操作：
  源「宿舍A-1」，点【HTTP】

现象：成功，[HTTP 200]，路径到 web 服务器

操作：
  源「访客-1」，点【HTTP】

现象：失败，访客 WiFi 出不了岛

操作：
  VPN 开，源「访客-3」，点【HTTP】→ 成功
  VPN 开，源「访客-3」，点【FTP】→ 成功（下载 share.txt）

口述：不止 Ping，还能演示真实 Web/FTP 业务；ACL 对应用层同样生效。


--------------------------------------------------------------------------------
第 6 步  保密区禁止外出（创新点）
--------------------------------------------------------------------------------

操作：
  源「人事处-1」，目标选 VPN 相关——Ping「VPN服务器」
  或人事处访问外网节点（若列表无 ext，Ping vpn_srv 即可说明 DMZ 方向）

现象：✕ 在 cs1 侧，阻断

再测：财务处-1 → VPN服务器，同样阻断

口述：保密区单向防护——外面进不来，里面也出不去 DMZ/外网。


--------------------------------------------------------------------------------
第 7 步  分校区互通
--------------------------------------------------------------------------------

操作：点左下【分校区】按钮

现象：拓扑切换为 cs2、校区2 主机 c2h1/c2h2；cs1—cs2 青色专线

操作：切回【主校区】，源「校区2-1」(c2h1)，目标「办公楼-1」，Ping

现象：成功，路径经 cs2 → cs1 → 主校区

口述：双校区通过核心专线互联，跨校区也要过 cs1。

第 8 步  ACL 可视化按钮

操作：点【ACL】

现象：多条链路同时高亮，展示防火墙关注的路径

操作：点【清除】，再做一个 Ping 看单条路径高亮

口述：规则集中在 cs1，前端把抽象 ACL 画到具体链路上，方便非专业人士理解。

================================================================================
第八部分  本项目创新点与关键点（答辩收尾用）
================================================================================

1. 三层架构 + 多 VLAN 校园网真实建模
   核心/汇聚/接入分层，区域划分贴近真实高校网络。

2. ACL 集中在核心 cs1 统一执行
   汇聚/接入只做转发；前端把阻断位置画在 sw13、as1、cs1 不同出口，体现多种机制。

3. 三类访客差异化设计
   WiFi 岛物理隔离（sw13 无上联）+ DMZ 访客（gst3）+ GRE VPN 接入，不是「一刀切访客区」。

4. DMZ 区可视化
   sw15 天蓝虚线直连 cs1；gst3 与 vpn_srv 隧道用靛蓝虚线，拓扑上一眼能看懂安全域。

5. VPN 开关只控 gst3，且「连上 VPN ≠ 全网权限」
   仍阻断人事/财务，体现最小权限原则。

6. 不止 Ping：HTTP / FTP 应用层演示
   从网络层到应用层完整闭环。

7. D3.js 动态路径高亮 + 阻断 ✕ 标记
   成功绿线、失败红线+叉号，非网络专业评委也能看懂「包走到哪里被踢掉」。

8. 双校区拓扑一键切换
   主校区 / 分校区分页展示，跨校区路径经 cs1—cs2 专线。

9. 前后端可选联调
   同一页面：连 Mininet 是真测（[Mininet] 标记），离线是本地模拟（演示就绪），答辩不依赖现场网络稳定性。

10. API 联调日志透明
    请求、探测、VPN 同步过程可见，体现工程化而不是纯 PPT 演示。
